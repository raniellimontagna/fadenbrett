## Codebase Patterns
- Monorepo with pnpm workspaces: root `package.json` delegates to `apps/web` via `--filter @fadenbrett/web`
- Quality checks: `pnpm type-check`, `pnpm lint`, `pnpm build`
- Frontend: React 19 + Vite 7 + TypeScript + Tailwind v4 (via @tailwindcss/vite plugin)
- Canvas engine: @xyflow/react (React Flow) with Zustand for state
- Feature folders: `src/features/<feature>/` for components, `src/store/` for Zustand stores
- Tailwind v4: Use `@theme` block in index.css for custom tokens (no tailwind.config.js)
- Dark theme by default: custom colors under `--color-fadenbrett-*`
- React Flow typed node data: cast to `Record<string, unknown>` when storing, cast back to specific type when reading in node components
- Custom nodeTypes object must be defined outside the component (or useMemo) to prevent re-mount cycles
- React Flow `onDoubleClick` fires for ALL elements (pane, nodes, edges). No `onPaneDoubleClick` prop exists — guard with `target.closest('.react-flow__node')`
- S023: `better-sqlite3` is a native module — requires `node-gyp rebuild` after `pnpm install` if bindings aren't prebuilt for the current Node version; run `node-gyp rebuild` in the better-sqlite3 package dir
- S023: Use raw SQL in `runMigrations()` (CREATE TABLE IF NOT EXISTS) instead of drizzle-kit at runtime — simpler and zero extra deps
- S023: API routes under `/api` prefix registered via Fastify scoped plugin; route files export a `FastifyInstance => Promise<void>` function
- S023: `DB_PATH=':memory:'` env var in tests to avoid writing to filesystem during vitest runs
- S024: Remove `persist` middleware entirely for API-backed stores; use lightweight `lsGet`/`lsSet` helpers for preferences only (theme, user, presentation stops)
- S024: Debounced autosave: use a `useBoardStore.subscribe()` at bottom of file that calls `debouncedApiSave` on nodes/edges reference change
- S025: WebSocket route: `{ websocket: true }` option + scoped plugin under `/ws` prefix; pure rebroadcast server — no state stored server-side; `@types/ws` required as dev dep for collab.ts typing
- S026: `@fastify/multipart` file upload: use `req.file()` + `for await` chunk collection; never use `apiFetch` for multipart (it overrides Content-Type); `@fastify/static` for serving uploads static dir
- S035: For incremental DB migrations, use `PRAGMA table_info('table')` to check existing columns, then `ALTER TABLE ADD COLUMN` for new ones — keeps backward compat with existing DBs
- S027: Docker multi-stage for api requires `python3 make g++` for better-sqlite3 native bindings even in runner stage; `VITE_API_URL=""` needs WS URL derived from `window.location` since WebSocket requires absolute URL
- S028: GitHub Actions with pnpm — `pnpm/action-setup@v4` must come before `actions/setup-node@v4`; `permissions: packages: write` needed for ghcr.io push with GITHUB_TOKEN
- S029: Proxmox LXC scripts — `features: nesting=1` required for Docker inside unprivileged LXC; use `pvesh get /cluster/nextid` for CT_ID; `pveam available --section system | sort -V` to get latest template
- Filter dimming: derive `displayNodes`/`displayEdges` in the canvas component (useMemo) by spreading style.opacity; never mutate store nodes for presentational state
- Zustand persist: use `createJSONStorage(() => adapter)` not raw `StateStorage`; cast `partialize` return as `unknown as S`; debounce the raw adapter's `setItem` to avoid drag thrash
- FilterBar and other store-only components don't need ReactFlowProvider; only components using `useReactFlow()` hooks do
- Export: `html-to-image` `toPng`/`toJpeg` on the root canvas div; filter out overlay elements with `data-export-exclude="true"`
- Note nodes use inline editing (double-click) instead of modal forms — store.updateNote called directly from the node component
- Shift+double-click on pane creates a note; plain double-click creates a card
- Custom edgeTypes object must be defined outside component (same as nodeTypes) to prevent re-mount cycles
- Connections use `type: 'yarn'` edge type; data stored as ConnectionData cast to Record<string, unknown>
- Double-click edge to edit connection properties (label, style, color)
- React Flow lasso: `selectionOnDrag` + `panOnDrag={[0, 1]}` (middle+left button pan) enables drag-to-select
- React Flow `onPaneContextMenu` callback type is `MouseEvent | React.MouseEvent` (union), not just `React.MouseEvent`
- Delete confirmation: show for multi-item (>1) deletions, single items delete immediately
- Multi-board: `partialize` must snapshot active board into `boards` map; `onRehydrateStorage` restores `nodes`/`edges` from the active board entry
- `useBoardStore.setState(fn)` works outside React components — useful for import handlers
- Compose multiple opacity sources (filters + search) into one final value in `displayNodes`/`displayEdges`
- S014: Store image as base64 DataURL in card data field — simplest browser-only approach; validate size (2MB) and MIME type client-side before FileReader
- S015: Undo/redo via `past/future` snapshot arrays in store; `pushHistory()` before every mutation; for `onNodesChange`, push only when a drag ends (changes with `type === 'position' && dragging === false`); keep `past/future` out of `partialize` (ephemeral)
- S016: Runtime theme switching via `[data-theme="X"]` CSS selectors overriding `--color-fadenbrett-*` vars; set `document.documentElement.setAttribute('data-theme', t)` in store action and in `onRehydrateStorage`
- S017: Frame nodes use React Flow `parentId` for automatic child-position relative coords; frame `zIndex: -1` renders behind other nodes; `NodeResizer` from `@xyflow/react`; ungroup by restoring absolute positions (add frame.position to children)
- S018: Presentation mode: store `presentationStops[]` + `presentationActive/Index` in Zustand, persisted; `fitView({ nodes: [{ id }], duration: 700 })` to zoom to stop; disable editing with `nodesDraggable/nodesConnectable/elementsSelectable={!presentationActive}` on ReactFlow; keyboard (ArrowLeft/Right/Esc) in `useEffect` on `window`
- S019: BroadcastChannel for cross-tab collab; `useBoardStore.subscribe()` outside the store to auto-broadcast nodes/edges on change (zero-invasive); BroadcastChannel does NOT echo to sender tab so no loop; store `localUser` (persisted) + ephemeral `remoteUsers`; `flowToScreenPosition` to convert flow coords to CSS cursor position
- S020: Templates stored as static TS arrays of `Node[]`+`Edge[]`; `applyTemplate(name, nodes, edges)` in store creates new board and switches to it; gallery is a modal with grid of cards
- S021: Long-press (touch-hold 600ms) shows context menu on mobile via `useLongPress` hook; `style={{ touchAction: 'none' }}` on canvas container prevents scroll conflicts; `panOnScroll={false}` avoids scroll-pan collision; context menu items have `min-h-[44px]` on mobile for Apple HIG touch targets
- S022: Mobile sidebar overlay pattern: `absolute inset-y-0 left-0 z-50 flex md:relative md:z-auto` (open) + `hidden md:flex` (closed); bottom-sheet modals use `items-end` on backdrop + `rounded-t-2xl w-full` on content; prefer `svh` over `vh` for mobile full-screen heights; ReactFlow MiniMap hidden with `!hidden sm:!block` className
- `replace_string_in_file` starting from the import line of a file appends instead of replacing the full file — use a unique suffix near the end of old content to avoid duplicate declarations
